# Contributing

Contributions are welcome and needed.

This project exists because people invest their time, ideas, and experience. If you are using this extension, you are already in the best position to improve it.

## How you can contribute

There are many ways to contribute, not just code:

- Report bugs and edge cases
- Improve documentation or clarify unclear parts
- Suggest features or improvements
- Submit pull requests
- Share real-world use cases and feedback

## Before you start

- Check existing issues and pull requests to avoid duplicates
- Open an issue first for larger changes to align on direction
- Keep changes focused and minimal

## Pull Requests

- Write clear, concise commit messages
- Follow the existing code style and structure
- Add tests where appropriate
- Ensure existing tests pass

## Mindset

Good contributions are not about perfection. They are about progress.

If something feels off, incomplete, or missing, it probably is. Trust that instinct and open an issue or PR.

## Why it matters

This project is actively maintained, but not fully funded. Contributions - whether code, feedback, or support - directly impact its quality, stability, and future.

If you rely on this extension, contributing back is the most effective way to keep it evolving.
